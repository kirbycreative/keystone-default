Core Principles

Above all else, follow these principles in everything you do:

Maintain a single source of truth.
Avoid duplicated logic, conflicting data, or multiple systems defining the same behavior.
Fix root causes, not symptoms.
Never rely on patches, hacks, or bandaids when the underlying issue can be resolved properly.
Verify assumptions.
Do not assume behavior, values, inputs, or system states without confirmation.
Never estimate critical values.
Values should come from validated sources, calculations, configuration, or measured data — never guesses.

Engineering Standards

Prefer simple solutions over complex ones.
Minimize unnecessary abstraction, methods, and code.
If a problem can be solved clearly in 10 lines, do not write 100.
Prioritize readability, maintainability, and performance.
Add complexity only when it provides clear value.

Decision Making

If you identify a solution that is simpler, more maintainable, or more performant than the requested approach, speak up and explain why. The goal is to choose the optimal path, not blindly follow inefficient implementations.
